# Authentication Guide for AI Agents

## Overview

This Django webapp uses **django-allauth** for authentication with support for:
- Email/password authentication
- OAuth (GitHub)
- Session-based authentication for web
- Token-based authentication for API (REST framework)

## Authentication Flow

### Web Application

1. **Login Process**:
   - Users access `/accounts/login/`
   - Authentication handled by django-allauth
   - After successful login, redirected to dashboard

2. **Session Management**:
   - Sessions are Django's standard session framework
   - LoginRequiredMixin protects views requiring authentication
   - Organization context stored in session (see Organization Management)

3. **Logout**:
   - `/accounts/logout/` (no confirmation page, immediate logout)
   - Configured with `ACCOUNT_LOGOUT_ON_GET = True`

### API Authentication

- Uses Django REST Framework Token Authentication
- Tokens generated via dj-rest-auth endpoints:
  - `/api/auth/registration/` - Register new user
  - `/api/auth/login/` - Get authentication token
  - Include token in headers: `Authorization: Token <token>`

## User Model

- Uses Django's default User model
- Additional user data linked via OrganizationMember (see Permissions)

## Key Settings

```python
# Authentication backends
AUTHENTICATION_BACKENDS = [
    'django.contrib.auth.backends.ModelBackend',
    'allauth.account.auth_backends.AuthenticationBackend',
]

# Allauth configuration
ACCOUNT_LOGIN_METHODS = ['email']
ACCOUNT_EMAIL_VERIFICATION = 'optional'
ACCOUNT_LOGOUT_ON_GET = True
```

## Protecting Views

### For Web Views

```python
from django.contrib.auth.mixins import LoginRequiredMixin

class MyView(LoginRequiredMixin, View):
    # View code here
    pass
```

### For API Views

```python
from rest_framework.permissions import IsAuthenticated

class MyAPIView(APIView):
    permission_classes = [IsAuthenticated]
```

## Testing Authentication

### In Tests

```python
from organization.tests.factories import UserFactory

def test_authenticated_view(client):
    user = UserFactory()
    client.login(username=user.username, password='password')
    response = client.get('/protected-url/')
    assert response.status_code == 200
```

### In Fixtures

```python
@pytest.fixture
def login_user(client):
    user = User.objects.create_user(username="user", password="password")
    client.login(username="user", password="password")
    return user
```

## Security Considerations

1. **Password Requirements**: 
   - Django's standard validators apply
   - Minimum 8 characters recommended

2. **Session Security**:
   - Sessions expire based on Django settings
   - CSRF protection enabled for all forms

3. **API Tokens**:
   - Tokens should be kept secret
   - No expiration by default (consider implementing if needed)

## Common Patterns

### Redirect After Login

```python
LOGIN_REDIRECT_URL = '/'  # Configured in settings
```

### Get Current User

In views:
```python
def my_view(request):
    current_user = request.user
    if request.user.is_authenticated:
        # User logic
        pass
```

In templates:
```django
{% if user.is_authenticated %}
    <p>Welcome {{ user.username }}!</p>
{% endif %}
```

## Troubleshooting

- **"This account is inactive"**: Check `user.is_active = True`
- **Login loop**: Check LOGIN_URL and LOGIN_REDIRECT_URL settings
- **API 401 Unauthorized**: Verify token is included in Authorization header
- **CSRF failures**: Ensure {% csrf_token %} in forms
